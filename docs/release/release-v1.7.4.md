# BiliBili 动态推送 Bot v1.7.4 发布说明

**版本号**：v1.7.4  
**发布日期**：2026-03-18  
**发布类型**：补丁版本 / 稳定发布  
**状态**：Release

## 发布说明

`v1.7.4` 是基于 `v1.7.3` 的补丁更新，重点围绕会话级主题色的渐变渲染一致性、绘图资源生命周期管理和字体初始化稳定性进行优化。本次更新让多色主题色输入、缓存刷新、链接解析卡片渲染和长期运行场景下的绘图行为更加稳定。

如果你之前已经为同一 UID 绑定了会话级主题色，本次升级会在启动时自动规范化已有颜色数据，一般不需要手工迁移。建议升级后重点关注多色渐变卡片、链接解析卡片、长时间运行后的绘图稳定性，以及自定义字体目录加载是否符合预期。

## 本次重点变更

### 1. 会话级主题色引入主题感知渐变渲染与统一规范化

本次更新重新整理了会话级主题色绑定的输入、存储和渲染方式：

- `/bili color` 和 `/bili config color` 的颜色输入会统一规范为稳定格式，例如小写十六进制和分号分隔
- 单色和多色输入都会生成更稳定的渐变调色板，尽量保留原始色相关系
- 渐变输入最多支持 4 个颜色停止点，避免颜色组合过多导致渲染不稳定
- 已有会话级颜色绑定会在启动时自动扫描并规范化，减少旧数据与新缓存键不一致的问题

这部分主要提升多色主题色在不同卡片上的一致性，减少“同样的颜色设置，最终渲染效果不稳定”的情况。

### 2. 动态、直播与链接解析卡片的渐变缓存和刷新行为更一致

围绕新的主题色机制，本次也统一了绘图缓存的处理方式：

- 动态、直播、链接解析卡片都会使用规范化后的颜色参与缓存键计算
- 修改会话级主题色后，会清理对应会话下的相关绘图缓存，减少继续复用旧底图的情况
- 链接解析卡片也接入同一套渐变调色板和作用域缓存逻辑，不同会话下的颜色表现更一致

这部分主要解决主题色修改后缓存未及时刷新，或者不同绘图入口颜色行为不一致的问题。

### 3. Skia 与字体资源生命周期优化，长时间运行更稳

本次继续整理图片渲染链路中的资源释放行为：

- 绘图过程中使用的 Paint、Shader、SVG 等 Skia 资源改为统一托管释放
- 字体样式匹配过程中不再持有临时字体样式集资源，减少 native 资源泄漏
- 在频繁生成动态卡片、直播卡片和链接解析卡片时，长期运行的稳定性更好

这部分主要面向长期运行实例，目标是减少绘图链路积累资源后带来的内存压力和偶发异常。

### 4. 字体目录初始化更稳健，首次启动和干净环境更不容易失败

针对字体加载初始化流程，本次补上了目录创建的边界情况：

- `data/font` 目录现在支持递归创建
- 在全新环境、目录层级缺失或自定义字体目录尚未建立时，字体加载更不容易因为目录不存在而失败
- 需要放置自定义字体的部署场景会更直接，不必先手工补齐每一级目录

这部分主要改善首次部署、迁移部署和干净环境下的启动体验。

## 反馈时建议附带的信息

如果你在升级到 `v1.7.4` 后发现问题，建议尽量附带以下信息：

- 使用版本：`v1.7.4`
- 部署方式：JAR / Docker / distribution
- 运行环境：Windows / Linux，JDK 版本，NapCat 版本
- 是否使用了会话级主题色绑定，具体颜色配置内容是什么
- 问题是否出现在多色渐变卡片、链接解析卡片、长时间运行后的绘图，或自定义字体加载场景
- 相关动态链接、直播间链接、日志、报错信息或复现步骤
