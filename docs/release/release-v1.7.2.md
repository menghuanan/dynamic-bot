# BiliBili 动态推送 Bot v1.7.2 发布说明

**版本号**：v1.7.2  
**发布日期**：2026-03-13  
**发布类型**：补丁版本 / 稳定发布  
**状态**：Release  

## 发布说明

`v1.7.2` 是基于 `v1.7.1` 的稳定更新，重点优化动态卡片可读性、订阅名称同步体验，以及 NapCat 消息日志的可读性。

本次更新不涉及新的配置迁移要求。如果你准备从 `v1.7.1` 升级到 `v1.7.2`，建议仍然先备份 `config/`、`data/`、`logs/`，并在升级后重点观察动态卡片显示效果和消息日志输出。

## 本次重点变更

### 1. 动态图片卡片新增媒体标签，并补齐视觉细节优化

本次更新为动态图片内容补充了更直观的媒体标签展示：

- GIF 图片会标记为 `动图`
- 超长图片会标记为 `长图`
- 标签文本的垂直居中效果已修正，避免出现视觉偏移
- 动态屏蔽提示文字颜色已调整为更适合阅读的中灰色

这部分调整主要用于提升动态卡片的信息辨识度，让特殊图片类型和状态提示更容易一眼看清。

### 2. 订阅名称会自动跟随作者最新昵称同步

当订阅作者修改昵称后，系统会在后续处理新动态时自动刷新本地订阅名称：

- 已存在的订阅项会跟随作者最新昵称更新
- 无需通过取消订阅再重新订阅来修正名称
- 长期运行实例中的订阅显示更容易保持最新状态

### 3. NapCat 消息日志预览更简洁，更适合排查问题

本次也对 NapCat 相关日志做了整理，减少冗长原始消息结构对阅读的干扰：

- 常见 CQ 码和消息段会以更直观的占位文本显示
- 混合消息的出站日志预览会更容易看出文本、图片和其他消息类型
- 排查收发消息问题时，可以更快聚焦到实际内容而不是大段原始负载

## 建议重点验证

1. 动态卡片显示

- GIF 图片是否显示 `动图`
- 长图是否显示 `长图`
- 标签文字是否保持垂直居中
- 屏蔽提示文字颜色是否符合预期

2. 订阅名称同步

- 作者改名后，下一次动态推送后订阅名称是否自动更新
- 老订阅数据是否仍能正常推送

3. NapCat 日志输出

- 收到带 CQ 码的消息时，日志是否更易读
- 发送混合消息时，日志预览是否能直观看出消息类型
