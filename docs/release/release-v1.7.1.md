# BiliBili 动态推送 Bot v1.7.1 发布说明

**版本号**：v1.7.1  
**发布日期**：2026-03-07  
**发布类型**：补丁版本 / 稳定发布  
**状态**：Release  

## 发布说明

`v1.7.1` 是基于 `v1.7` 测试版后的整理性补丁发布，重点处理编码一致性、中文乱码与版本信息同步问题。

本次发布的目标不是继续扩大业务改动，而是收敛已经暴露出来的编码风险，确保源码、配置读写、日志输出与版本标识在常见运行方式下更加稳定可控。

如果你准备从 `v1.7` 升级到 `v1.7.1`，建议仍然先备份 `config/`、`data/`、`logs/`，再完成一次小范围灰度验证。

## 本次重点变更

### 1. 统一项目文本编码为 UTF-8 without BOM

本次对受版本控制的项目文本文件做了编码一致性整理：

- 清理仓库内已确认存在的 `UTF-8 with BOM` 文件
- 新增编码回归测试，防止后续再次引入 BOM
- 在 `.editorconfig` 中补充 `charset = utf-8` 约束

这部分调整的目标是减少跨编辑器、跨系统下的编码歧义，降低后续再次出现乱码的概率。

### 2. 修复已污染的中文文本与默认编码 IO

本次补丁修复了之前已经进入源码的中文乱码文本，并补齐了少数编码敏感 IO 的显式字符集：

- 修复任务资源策略说明中的乱码文本
- 修复临时 JSON 工具中的乱码日志文案
- 修复 `SendTasker` 中残留的乱码注释
- 将翻译 HTTP 响应读取改为显式 `UTF-8`
- 将守护进程日志写入改为显式 `UTF-8`

本次改动不涉及业务逻辑变更，主要是保证中文内容在读取、写入和源码维护过程中不再被错误编码破坏。

### 3. 版本与文档同步更新到 v1.7.1

本次会话中已同步更新：

- Gradle 默认版本号
- Docker Compose 默认镜像标签
- 程序帮助信息中的版本显示
- README 中的当前 JAR 文件名、版本标签与运行示例

README 顶部主标题本次按约定保持不变，没有作为版本同步对象修改。

## 建议重点验证

建议升级后优先验证以下内容：

1. 中文配置与数据读写

- 手工编辑 `BiliConfig.yml`、`BiliData.yml`、`bot.yml` 后，重启加载是否正常
- 含中文的订阅名、模板内容、群组名、管理员信息是否能正确显示
- 配置保存后重新打开文件，是否仍保持正常中文

2. 编码敏感链路

- 翻译功能开启后，返回内容是否存在乱码
- 守护进程日志 `logs/daemon/Daemon_YYYY-MM-DD.log` 是否保持正常中文
- JSON 异常样本输出与日志提示是否能正确显示中文

3. 版本与部署入口

- JAR 构建产物文件名是否为 `dynamic-bot-1.7.1.jar`
- Docker Compose 默认镜像标签是否已切换到 `1.7.1`
- `--help` 帮助输出中的版本号是否为 `v1.7.1`

## 已完成的基础验证

在本次会话中，已完成以下基础验证：

- 受版本控制的项目文本文件 BOM 复查为空
- 编码回归测试 `EncodingRegressionGuardTest` 已通过
- `./gradlew.bat test -q` 已成功执行
- 编码相关源码替换后未发现新的测试回归
- 项目内当前版本入口已同步更新到 `1.7.1`

需要注意的是，这些验证主要覆盖编码一致性与基础回归，仍不能完全替代真实运行环境中的功能验证。
