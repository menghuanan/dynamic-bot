# BiliBili 动态推送 Bot v1.7 测试版发布说明

**版本号**：v1.7  
**发布日期**：2026-03-07  
**发布类型**：测试版 / 预发布  
**状态**：Pre-release  

## 测试版说明

本版本先以测试版形式发布。

当前参与测试的人数较少，暂时无法确认项目在所有运行环境、配置组合、消息链路与使用场景下都表现正常。因此本次发布的目标，是先扩大真实环境覆盖范围，尽早发现仍可能存在的兼容性问题、命令行为问题、渲染异常或部署问题。

如果你准备用于生产环境，建议先完成以下操作：

- 先备份 `config/`、`data/`、`logs/`
- 先在小范围群聊或测试账号中灰度验证
- 优先观察图片渲染、命令响应、NapCat 连接和 Docker 运行稳定性

## 本次重点变更

### 1. 命令管线与生命周期管理重构

本版本对 Bot 的命令处理链路与运行期职责做了较大规模重构：

- 拆分命令路由、分发与权限处理逻辑
- 引入更清晰的消息网关、事件分发与生命周期管理结构
- 为订阅、群组、过滤器、模板、管理员等命令能力补齐独立服务层
- 补充多项回归测试，降低重构后的行为回退风险

这部分改动范围较大，是本次测试版最需要重点覆盖的区域之一。

### 2. 支持会话级动态卡片主题色绑定

新增“按会话绑定主题色”的能力，同一 UID 在不同群聊或私聊中可以使用不同的动态卡片主题色。

相关变化包括：

- 颜色设置命令 `/bili config color` 改为在当前会话内生效
- 动态、直播、链接解析等渲染逻辑改为读取会话级颜色配置
- 旧的全局颜色字段迁移到新的会话级绑定结构
- 数据版本提升到 2，用于兼容旧数据迁移

如果你之前依赖全局颜色设置，建议重点确认迁移后的显示结果是否符合预期。

### 3. 字体加载策略调整，并补充回归测试

字体加载逻辑已进一步收紧：

- 移除了自动下载网络字体的行为
- 保留本地字体扫描、用户覆盖字体和内置回退字体加载
- 新增字体加载回归测试，确保后续不会重新引入自动下载逻辑

这项调整的目标是降低启动时的外部依赖和不确定性，避免因为网络环境导致字体行为不可控。

### 4. 构建与分发包逻辑精简

本次也对构建与仓库结构做了精简整理：

- 删除手工维护的旧启动脚本与历史 PowerShell 自动化脚本
- 保留 Gradle distribution 打包能力
- 将启动脚本改为在构建时生成到 `build/generated/distribution-scripts`
- 分发包继续保留 `bin/start.bat`、`bin/start.sh` 与可运行 JAR

这部分调整主要是为了减少仓库中的冗余脚本，避免源码树与构建产物混杂。

### 5. 版本与文档同步更新到 v1.7

本次会话中已同步更新：

- Gradle 默认版本号
- Docker Compose 默认镜像标签
- 程序启动帮助信息
- README 中的 JAR 文件名、版本标签与运行示例

## 建议重点测试

建议测试人员优先覆盖以下内容：

1. 命令可用性

- 常用 `/bili` 命令是否仍能正确响应
- 管理员、过滤器、模板、分组、黑名单等子命令是否存在权限或路由异常
- 快捷命令与群聊限定命令是否行为正常

2. 颜色与渲染

- `/bili config color` 在不同群聊、私聊中是否可以分别生效
- 动态卡片、直播卡片、链接解析卡片的颜色是否符合当前会话配置
- 老数据升级后是否出现颜色丢失或错配

3. 图片发送链路

- NapCat 图片发送在不同模式下是否正常
- 动态、直播、帮助图、链接解析图是否能稳定发送
- 大图、长图和异常渲染场景下是否存在发送失败或卡死

4. 部署与运行稳定性

- JAR 直接运行是否正常
- Docker 部署、重启、长期运行是否稳定
- distribution 产物解压后是否可以直接启动
