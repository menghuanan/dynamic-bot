# BiliBili 动态推送 Bot v1.8.6 发布说明

**版本号**：v1.8.6
**发布日期**：2026-04-18

---

## 变更摘要

主要围绕**内存管理、启动脚本一致性以及发行方式完善**进行补充优化。

* 统一裸机与 Docker 内存基线配置
* 引入 jemalloc 注入机制，降低内存漂移
* 优化 JVM 空闲期内存回收策略
* 发布平台化启动脚本与发行包
* 修复部分边界逻辑与测试

---

## 详细变更说明

### 1. 内存管理优化（重点）

针对长时间运行场景，增强 JVM 内存回收能力：

* 新增 G1 周期性 GC 参数：

  * `-XX:G1PeriodicGCInterval=60000`（每分钟触发）
  * `-XX:G1PeriodicGCSystemLoadThreshold=0`
* 调整堆收缩策略：

  * `-XX:MinHeapFreeRatio=10`
  * `-XX:MaxHeapFreeRatio=20`

👉 在空闲期更积极释放堆内存，减少内存占用长期膨胀问题。

---

### 2. jemalloc 注入支持（Linux）

为降低内存碎片与漂移问题：

* 在 Linux 启动脚本中注入 jemalloc（通过 LD_PRELOAD）
* 确保在 JVM 启动前生效
* 避免回退到 glibc malloc

同时：

* Windows 启动脚本明确说明不支持该机制（无 LD_PRELOAD）

👉 提升 Linux 环境下的内存稳定性（尤其是长时间运行）。

---

### 3. 启动脚本与 Docker 配置统一

统一裸机发行与 Docker 的默认内存基线：

* 默认 JVM 参数调整为：

  * `-Xms64m -Xmx160m`

替代旧配置：

* `-Xms512m -Xmx2g`

同时：

* 更新 README，说明启动脚本使用方式
* 明确与直接运行 JAR 的差异

👉 避免不同部署方式下的资源配置不一致问题。

---

### 4. 发行与启动方式完善

* 发布平台化发行包（release archives）
* 新增平台启动引导（launcher startup）

👉 提供更标准化的部署入口，降低使用门槛。

---

### 5. 修复与测试

* 修复 AtAllService 中“全部动态”与细分类型的存储规范化问题
* 新增 / 更新回归测试：

  * 验证 jemalloc 注入逻辑
  * 校验 Docker 与裸机配置一致性

---

## 影响说明

本次更新主要为运行时优化：

* 默认内存配置发生变化（更保守）
* 建议根据实际负载自行调整 JVM 参数

👉 对低内存环境更友好，但高负载场景建议手动调优。

本次更新主要集中在**运行时稳定性与部署一致性**。

如果你运行在 Docker 或 NAS / 低内存环境，建议使用本版本默认配置；如有更高性能需求，可在此基础上进一步调优 JVM 参数。
