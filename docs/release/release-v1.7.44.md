# BiliBili 动态推送 Bot v1.7.44 发布说明

**版本号**：v1.7.44  
**发布日期**：2026-03-25  
**发布类型**：常规更新  
**状态**：Release  

---

## 变更摘要

本次更新主要聚焦于 Docker 运行环境优化与链接解析能力增强，同时进一步完善动态渲染逻辑。通过引入软件渲染模式与精简镜像体积，显著降低资源占用，并新增对 opus 链接的完整支持。

* Docker 迁移至纯软件渲染模式，优化内存占用
* 基础镜像由 JDK 精简为 JRE，减小镜像体积
* 新增 opus 链接解析能力，优化专栏动态展示
* 完善动态结构处理逻辑与链接规范化

---

## 详细变更说明

### 1. Docker 渲染架构重构（重要）

对 Docker 运行环境进行重构，迁移至纯软件渲染模式：

* 移除 Xvfb 及相关 X11 依赖
* 改为使用 Skiko 软件渲染
* 更新启动脚本：

  * 移除 Xvfb 启动与清理逻辑
* 精简 Dockerfile：

  * 删除不必要的系统库依赖

内存参数优化：

* 调整 JVM 参数：

  * MaxDirectMemorySize
  * Xss

👉 在保证渲染能力的前提下，降低整体资源占用并提升运行稳定性（尤其适用于 Docker / NAS 场景）。

---

### 2. 构建优化（镜像体积缩减）

优化镜像构建策略：

* 基础镜像由 `eclipse-temurin:17-jdk`
  → 替换为 `eclipse-temurin:17-jre`

原因：

* 项目以已编译的可执行 JAR 运行
* 无需 JDK 编译工具链

👉 显著减小镜像体积，加快拉取与部署速度。

---

### 3. 链接解析能力增强（opus 支持）

新增对 opus 链接的完整支持，并优化专栏相关逻辑：

* 新增数据结构：

  * ArticleViewInfo（用于获取专栏对应动态 ID）
* 新增链接类型：

  * LinkType.OpusWithCv
  * 用于区分 opus 与普通专栏链接

解析逻辑优化：

* 为 opus 链接添加专用解析流程

  * 避免降级使用文章详情 API
* 更新链接转换逻辑：

  * 正确处理 opus 与 cv 链接格式

👉 提升解析准确性，减少错误匹配与信息丢失。

---

### 4. 动态展示逻辑优化

配合 opus 支持，对动态结构处理进行增强：

* 优化动态显示标准化逻辑
* 支持在 opus 动态中覆盖文章 ID
* 提升专栏内容在动态中的展示一致性

同时新增：

* 对应单元测试
* 回归测试保障

👉 解决专栏与动态混合场景下的显示异常问题。
