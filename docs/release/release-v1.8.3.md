# BiliBili 动态推送 Bot v1.8.3 发布说明

**版本号**：v1.8.3
**发布日期**：2026-03-28

---

## 变更摘要

本次为 **v1.8.3 稳定性优化更新**，主要围绕登录流程回退、资源管理优化以及适配器行为调整展开，进一步提升运行稳定性与长期运行表现。

* 恢复登录二维码文件发送链路
* 优化 OneBot11 本地图片处理策略（改为文件 URI）
* 修复 Skia 相关内存泄漏问题
* 完善二维码渲染器测试与资源管理
* 清理过时设计文档

---

## 详细变更说明

### 1. 登录流程调整（回退优化）

* 恢复登录二维码通过文件发送的实现方式
* 相较 v1.8.2 的纯二进制发送方案，本次回退为更稳定的实现路径

👉 在不同平台适配器下具有更好的兼容性与实际表现。

---

### 2. OneBot11 图片发送策略调整（重要）

优化适配器层对本地图片的处理方式：

* 不再将本地图片强制转换为 Base64
* 改为保留 `file://` URI 形式
* 由下游客户端自行决定：

  * 是否转为 Base64
  * 或直接使用文件发送

同时：

* 移除 encodeLocalImageFile / resolveLocalImageFile 方法
* 使用 ImageCache.toFileUrl() 统一生成文件 URI

👉 提升适配器通用性，兼容不同 OneBot11 实现（如 NapCat、LLBot 等）。

---

### 3. Skia 资源管理与内存优化

修复长时间运行下的内存泄漏问题：

* 新增 FontUtils.resetParagraphCache()

  * 显式清理全局段落缓存
* 在 SkiaManager 清理流程中加入缓存重置
* SkiaCleanupTasker 支持周期性清理

  * 不再仅依赖空闲触发

问题修复：

* 修复 QrCodeDraw 备用 Logo 分支字体未释放问题
* 明确 createImageWithSession 返回资源需由调用方释放

👉 显著改善长时间运行（7x24）场景下的内存稳定性。

---

### 4. 二维码渲染器测试优化

* 登录二维码渲染逻辑已迁移至 LoginQrCodeRenderer
* 更新回归测试：

  * 优先测试新渲染器
  * 若不存在则回退旧实现（兼容历史分支）

👉 保证新旧逻辑下测试均可正常运行。

---

### 5. 文档清理

* 删除已完成的登录重构设计文档
* 避免历史文档对当前实现造成误导

---

## 影响说明

本次更新为优化与修复版本：

* 不涉及配置结构变更
* 不影响现有功能使用
* 建议所有 v1.8.x 用户升级

如果你在升级到 v1.8.3 后发现问题，建议附带以下信息：

* 使用版本：v1.8.3
* 部署方式：JAR / Docker / distribution
* 使用平台：

* 是否涉及以下场景：

  * 登录二维码异常

  * 内存占用异常或持续增长
* 相关日志（尤其是错误日志）
* 可复现步骤
