# BiliBili 动态推送 Bot v1.7.3 发布说明

**版本号**：v1.7.3  
**发布日期**：2026-03-16  
**发布类型**：补丁版本 / 稳定发布  
**状态**：Release

## 发布说明

`v1.7.3` 是基于 `v1.7.2` 的补丁更新，重点修复多会话主题色下的图片缓存串用、PGC 卡片上下留白不平衡，以及关闭绘图时推送与链接解析的退化行为不完整等问题。同时，本次也收敛了 Docker 发布标签策略，后续正式发布只保留精确版本标签和 `latest`。

本次更新不涉及强制配置迁移。如果你在不同群聊或私聊中为同一 UID 配置了不同主题色，或者使用了关闭绘图、`DrawOnly` 模板、PGC 番剧卡片、群内链接解析等场景，建议升级后优先确认相关表现是否符合预期。

## 本次重点变更

### 1. 图片缓存按推送目标和主题色隔离，避免串图与串色

本次更新会根据推送目标和主题色生成独立的动态与直播绘图缓存：

- 同一 UID 或同一条内容在不同群聊、私聊中会使用各自独立的缓存图
- 同一推送目标切换不同主题色后，也会生成对应的新缓存
- 避免会话级主题色绑定场景下复用旧缓存，导致不同联系人收到错误配色的图片

这部分主要解决“同一订阅对象在不同会话里使用不同颜色，但最终推送图被旧缓存覆盖”的问题。

### 2. PGC 卡片上下安全留白重新平衡，减少贴边和底部失衡

针对番剧、电影、国创等 PGC 小卡片，本次连续修复了顶部安全间距和最终底部留白：

- 卡片顶部会为徽章和正文保留更稳定的安全距离
- 底部留白会与卡片侧边距更接近，减少视觉上“下边过厚”或“不对称”的情况
- 含简介等多行文本时，高度计算更准确，降低内容贴边或底部被挤压的问题

这部分改动主要提升 PGC 动态卡片的整体排版观感和一致性。

### 3. 关闭绘图时的退化行为和 `cutLine` 排版逻辑已补齐

为了让“关闭绘图”场景不再出现空消息或异常排版，本次补齐了多处退化逻辑：

- 关闭推送绘图时，含 `{draw}` 的模板会自动回退为文本标题和链接，而不是留下空白消息
- 关闭链接解析绘图时，群内链接解析会直接返回标准链接文本
- 翻译分隔线 `cutLine` 只会在实际存在译文时参与测量与渲染，避免无译文时出现多余分隔，同时让有译文时的排版高度更准确
- `@全体` 自动降级提醒和缓存清理连续失败告警会统一走通知开关，减少通知行为不一致的问题

这部分主要影响关闭绘图开关、模板预览、链接解析和管理员通知相关场景。

### 4. Docker 发布标签策略收敛为精确版本标签和 `latest`

围绕发布流程，本次同步调整了 Docker 镜像标签策略：

- GitHub Actions 不再自动发布 `v1.7` 这类可变的次要版本标签
- 正式发布只会产出精确版本标签（如 `v1.7.3`）和 `latest`
- 相关发布说明与回归测试已同步更新，减少镜像标签漂移带来的歧义

如果你之前依赖次要版本标签拉取镜像，升级后建议改为使用精确版本标签或 `latest`。

## 反馈时建议附带的信息

如果你在升级到 `v1.7.3` 后发现问题，建议尽量附带以下信息：

- 使用版本：`v1.7.3`
- 部署方式：JAR / Docker / distribution
- 运行环境：Windows / Linux，JDK 版本，NapCat 版本
- 是否在不同群聊或私聊中为同一 UID 配置了不同主题色，相关颜色配置或模板设置
- 问题是否出现在 PGC 卡片、关闭绘图、链接解析、Docker 标签拉取等场景
- 相关动态链接、直播间链接、日志、报错信息或复现步骤
